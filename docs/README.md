- Requisitos usam identificadores estáveis (`RF`, `RNF`, `SEG`, `UX`).
- Mudanças de escopo entram como issues e são priorizadas no GitHub Project.
- Evidências visuais ficam em `assets/screenshots/`; executáveis e saves não são versionados.
- Métricas envolvendo menores só podem ser coletadas com base legal, transparência e minimização compatíveis com a LGPD.
