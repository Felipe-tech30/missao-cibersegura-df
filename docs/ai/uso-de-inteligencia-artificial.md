# Uso responsável de inteligência artificial

## Identificação

O projeto foi desenvolvido com assistência do **OpenAI Codex, baseado no modelo GPT-5**. A ferramenta apoiou a engenharia e a documentação; não é componente executado dentro do jogo.

## Atividades apoiadas

- análise e modularização do código C/raylib;
- implementação e revisão das mecânicas educativas;
- criação dos elementos visuais com primitivas da raylib;
- redação inicial das orientações de prevenção;
- elaboração dos artefatos de arquitetura, requisitos, segurança, UX/UI e cloud;
- automação de compilação, verificação e apresentação.

## O que o jogo não faz

- não embarca modelo de linguagem nem chama API de IA;
- não envia progresso ou dados do jogador à OpenAI;
- não realiza perfilamento, reconhecimento ou decisão automatizada;
- não substitui orientação jurídica, psicológica, pedagógica ou policial;
- não afirma ser software oficial da PCDF.

## Supervisão humana

Antes de uso público, conteúdo e código devem ser revisados por pessoas com conhecimentos em segurança da informação, educação de adolescentes, legislação brasileira e comunicação institucional. O mantenedor aprova toda release.

| Risco | Controle proposto |
|---|---|
| Conteúdo incorreto ou desatualizado | Revisão especializada e data de revisão por release |
| Linguagem alarmista | Teste de compreensão e tom com o público-alvo |
| Código vulnerável sugerido pela IA | Revisão humana, compilação com avisos e análise estática |
| Confusão com produto oficial | Aviso no jogo, README e apresentação |
| Estereótipos ou exclusão | Revisão de diversidade, acessibilidade e teste de UX |

Os desenhos atuais são construídos em código. A captura de gameplay foi produzida pelo próprio executável. Não foram usados logotipos oficiais nem imagens externas sem rastreabilidade.
