# Como contribuir

1. Selecione uma issue com objetivo e aceite claros.
2. Crie `feat/`, `fix/`, `docs/` ou `security/` a partir de `main`.
3. Faça mudanças pequenas; compile e teste o fluxo afetado.
4. Abra PR vinculando a issue e anexando evidência de UI.

Definition of Done: aceite atendido; build sem novos avisos; documentação atualizada; nenhum segredo, save ou binário; revisão humana; segurança, privacidade e acessibilidade avaliadas.

Orientações sobre crimes cibernéticos precisam de validação especializada antes de publicação externa.
